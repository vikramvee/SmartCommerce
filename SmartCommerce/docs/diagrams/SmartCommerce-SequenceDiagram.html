Outbox Processor — Reliable Event Publishing

BackgroundService that polls DynamoDB for unpublished events and delivers them to SNS atomically. Guarantees no event loss even if SNS is temporarily unavailable.

(10s interval) participant Proc as OutboxProcessor participant DynOutbox as DynamoDB
Outbox Table participant SNS as SNS Topic
(smartcommerce-orders) participant SQS as SQS Queue loop Every 10 seconds Timer->>Proc: ExecuteAsync tick Proc->>DynOutbox: Query
PK = OUTBOX#UNPUBLISHED
Limit = 10 alt No messages DynOutbox-->>Proc: [] empty Note over Proc: Sleep 10s, retry else Messages found DynOutbox-->>Proc: [OutboxMessage, ...] loop For each message Proc->>Proc: Deserialize payload
→ IDomainEvent Proc->>SNS: Publish(payload, MessageAttributes) Note over SNS: MessageAttributes:
EventType = order.placed
TenantId = tenant-alpha alt SNS publish fails SNS-->>Proc: Exception Note over Proc: Log error, skip delete
Retry on next poll else SNS publish succeeds SNS-->>Proc: MessageId ✅ SNS->>SQS: Fanout to subscribers Proc->>DynOutbox: TransactWriteItems

SQS Event Consumer — Inbound Event Processing

BackgroundService that long-polls SQS, unwraps the SNS envelope, deserializes the domain event, and dispatches to the correct event handler. Delete-on-success guarantees no message loss.
